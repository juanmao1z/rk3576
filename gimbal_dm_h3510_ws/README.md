# DM-H3510 Gimbal Workspace

DM-H3510 云台控制工作区。这个目录用于后续放置云台通信、姿态控制、标定参数和与视觉检测链路联动的脚本。

## 是否需要新建工作区

需要，建议保留为独立工作区。

原因：

- 云台控制和相机/YOLO 推理是不同生命周期：云台需要独立初始化、归中、限位、停止和故障保护。
- DM-H3510 后续可能使用串口、CAN、PWM 或厂商 SDK，依赖和权限不应混进 YOLO 工作区。
- 视觉检测结果可以通过 ROS2 话题或轻量进程接口驱动云台，不需要把云台代码写进检测服务本体。
- 独立目录便于以后在开发板上部署为单独服务，也便于停止视觉服务时保留或安全关闭云台。

## 目录结构

```text
gimbal_dm_h3510_ws/
  config/           云台串口/CAN/限位/PID/归中参数
  docs/             DM-H3510 接线、协议、标定和联调记录
  scripts/
    board/          开发板侧启动、停止、诊断脚本
    windows/        Windows/ADB 侧部署、端口映射和调试入口
  src/              云台驱动、控制节点或协议适配代码
```

## 和现有工作区的边界

| 工作区 | 职责 |
| --- | --- |
| `camera_web_cpp_ws` | 摄像头采集和 MJPEG 原始图像流 |
| `drone_yolo_web_cpp_ws` | 无人机目标检测、Web 显示和 `/yolo/detections` 发布 |
| `gimbal_dm_h3510_ws` | DM-H3510 云台设备通信、姿态控制和跟踪指令执行 |

## 推荐集成方式

第一阶段先做云台单机控制：

1. 确认 DM-H3510 的通信方式、设备节点和供电接线。
2. 在 `config/` 记录串口/CAN 参数、角度限位、方向反转和归中位置。
3. 在 `scripts/board/` 添加最小启动/停止/归中脚本。
4. 在 `src/` 添加驱动或 ROS2 节点。

第二阶段再接入视觉闭环：

1. 继续让 YOLO 工作区发布统一检测结果 `/yolo/detections`。
2. 云台工作区单独订阅检测结果，计算目标中心偏差。
3. 云台工作区输出 pan/tilt 控制量，负责限幅、死区和丢目标回中。
4. 上层联动脚本再组合启动摄像头、YOLO 和云台，不合并三个工作区。

## 预留 ROS2 接口

建议后续保持这些接口方向：

| 接口 | 建议 |
| --- | --- |
| 输入检测 | 订阅 `/yolo/detections`，类型保持 `vision_msgs/msg/Detection2DArray` |
| 输入图像尺寸 | 复用相机参数或从检测节点附带配置读取 |
| 输出云台状态 | 发布 `/gimbal/state` |
| 输出控制指令 | 内部驱动接口或发布 `/gimbal/cmd` |

## 维护约定

- 不把 DM-H3510 驱动代码放入 `drone_yolo_web_cpp_ws`。
- 不在相机工作区里加入云台设备初始化逻辑。
- 联动启动脚本可以放在 `workspace\scripts\windows`，但具体云台逻辑放回本工作区。
- 拿到厂商协议或 SDK 后，先记录到 `docs/`，再实现 `src/`。
